Our Process

The Icalier Way.

We combine practices from Lean and Agile to ship a high quality products on time.

Section 1 Section 2 Section 3 Section 4 Section 5
1

Understanding.

Starting from a clear beginning, we let the whole team to clarify doubts about the problem, customer, business, users, value proposition, and how successfully the solution could be.

2

Definition.

Helping to build a shared comprenhension among stakeholders, the entire team is encouraged to live a collaborative design session to define how the solution should be built.

3

Design & Prototyping.

We look to create a non-functional prototype as fast as possible and move forward into coding. Additionally we will suggest a User Research session to be considered in order to learn more about the product before launching it to the masses.

4

Build.

The product should be coded at some point! We commit ourselves to constantly notify and show you all the progress along this module showing a working and functional product.

5

Measure & Learn.

What is not measured cannot be improved. With a set of metrics already defined we can learn how to improve the experience, function or value proposition, either adding features or removing waste.
